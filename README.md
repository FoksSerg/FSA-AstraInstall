## 📖 Описание проекта

**FSA-AstraInstall** — комплексная автоматизированная система для управления Linux-системами с **гибким функционалом установки различных Wine сборок**. Система обеспечивает автоматическое обновление операционных систем, установку и управление компонентами (Wine, среды разработки, приложения) с возможностью быстрого конфигурирования, создания новых образов, удаления существующих и переустановки в процессе эксплуатации.

### 🎯 Ключевые особенности

- 🍷 **Гибкое управление Wine сборками** — поддержка различных версий Wine и конфигураций
- ⚡ **Быстрое конфигурирование** — интуитивный интерфейс для настройки компонентов
- 🔄 **Динамическое управление экземплярами** — создание и управление множественными экземплярами Wine приложений
- 📦 **Умный выбор архивов** — автоматический поиск и диалог выбора при отсутствии компонентов
- 🎛️ **Централизованное управление состоянием** — синхронизация таймеров, мониторинга и блокировки кнопок
- 🗑️ **Безопасное удаление** — корректное удаление образов и связанных компонентов
- ⚙️ **Конфигурация компонентов** — все параметры (пути, имена папок, группы архивов) настраиваются через конфигурацию компонентов, без жестких привязок в коде

## 🍷 Гибкое управление Wine сборками

### Управление экземплярами Wine приложений

Система поддерживает **динамическое создание и управление экземплярами Wine приложений** для различных wineprefix. Это позволяет:

- 🎯 **Создавать множественные экземпляры** одного приложения для разных wineprefix
- 🔄 **Быстро переключаться** между различными конфигурациями
- 🗑️ **Безопасно удалять** экземпляры при удалении wineprefix
- 📦 **Использовать шаблоны** для быстрого создания новых экземпляров

**Пример использования:**
```python
# Создание экземпляра Notepad++ для wineprefix
instance_id, instance_data = wine_app_manager.get_or_create_instance(
    wineprefix_id='wineprefix',
    template_id='wine_app_notepad_plus_plus'
)
```

### Быстрое конфигурирование компонентов

Система предоставляет гибкие инструменты для конфигурирования компонентов:

- ⚙️ **PackageManager GUI** — визуальный редактор конфигураций компонентов
- 📝 **JSON конфигурации** — человекочитаемый формат конфигураций
- 🔗 **Управление зависимостями** — автоматическое разрешение зависимостей между компонентами
- 🎨 **Шаблоны компонентов** — быстрое создание новых компонентов на основе шаблонов

### Создание и удаление образов

- 🆕 **Создание образов** — быстрое создание новых образов Wine приложений из установленных компонентов
- 🗑️ **Удаление образов** — безопасное удаление с автоматической очисткой всех связанных экземпляров
- 📋 **Управление шаблонами** — создание и редактирование шаблонов для быстрого развертывания

### Умный поиск и выбор архивов

При установке компонента система автоматически:

1. 🔍 **Ищет архив** в стандартных местах (настраивается через `ASTRAPACK_DIR_NAME` и конфигурацию компонентов)
2. 📂 **Предлагает выбрать вручную** через диалог, если архив не найден
3. ⏸️ **Приостанавливает таймер** на время выбора файла
4. ✅ **Продолжает установку** после выбора архива

**Преимущества:**
- Не нужно прерывать установку при отсутствии архива
- Можно использовать архивы с другими именами
- Поддержка сетевых путей и USB-накопителей
- Автоматическое определение начальной директории
- Все пути и имена папок настраиваются через конфигурацию компонентов

### Конфигурация компонентов

Система использует централизованную конфигурацию компонентов (`COMPONENTS_CONFIG`), где все параметры настраиваются без жестких привязок в коде:

- **Имена папок** — `source_dir`, `winetricks_dir`, `wine_gecko_dir`, `default_template_output_dir`
- **Пути установки** — `wineprefix_path`, `template_output_dir`, `archive_output_dir`
- **Группы архивов** — `archive_group` (например, 'Cont', 'Astra') для организации архивов в подпапках
- **Приоритеты источников** — `source_priority` ('archive' или 'directory') для определения порядка поиска
- **Зависимости** — `dependencies` для автоматического разрешения зависимостей

**Преимущества:**
- Легко добавлять новые компоненты без изменения кода
- Единая точка конфигурации для всех параметров
- Гибкая настройка путей и имен для разных окружений

## 📦 Управление компонентами

### Система компонентов

Проект использует модульную систему компонентов, каждый из которых может быть:
- **Wine пакеты** — wine_astraregul, wine_9
- **Wine окружение** — WINEPREFIX с настройками
- **Winetricks библиотеки** — dotnet48, vcrun2013, vcrun2022

- **Приложения** — Astra.IDE, CONT-Designer
- **Ярлыки рабочего стола** — автоматическое создание ярлыков

### Управление экземплярами Wine приложений

Система поддерживает **динамическое создание и управление экземплярами** Wine приложений:

- 🎯 **Создание экземпляров** — автоматическое создание экземпляров приложений для различных wineprefix
- 🔄 **Управление зависимостями** — автоматическое разрешение зависимостей между экземплярами и wineprefix
- 🗑️ **Безопасное удаление** — при удалении wineprefix автоматически удаляются все связанные экземпляры
- 📋 **Шаблоны** — использование шаблонов для быстрого создания новых экземпляров

### PackageManager — инструмент управления

**Назначение:** Создание, управление и архивация компонентов.

**Основные функции:**
1. **Создание компонентов** — создание новых компонентов с конфигурацией
2. **Управление компонентами** — просмотр, редактирование, удаление
3. **Создание сборок** — формирование сборок из нескольких компонентов
4. **Архивация** — массовая архивация компонентов в tar.gz архивы
5. **Структура папок** — управление структурой архивов компонентов
6. **Создание шаблонов** — быстрое создание шаблонов Wine приложений

**Расположение архивов:** `AstraPack/`

**Формат компонентов:**
- Каждый компонент хранится в отдельном tar.gz архиве
- Архив содержит файлы компонента и конфигурацию (JSON)
- Конфигурация описывает тип компонента, зависимости, пути установки

### Умный поиск и выбор архивов

При установке компонента система:

1. 🔍 **Автоматически ищет архив** в стандартных местах (AstraPack/)

3. ⏸️ **Приостанавливает таймер** на время выбора файла
4. ✅ **Продолжает установку** после выбора архива

**Преимущества:**
- Не нужно прерывать установку при отсутствии архива
- Можно использовать архивы с другими именами
- Поддержка сетевых путей и USB-накопителей
- Автоматическое определение начальной директории (рекурсивный поиск существующей директории)
