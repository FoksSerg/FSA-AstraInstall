# Документация настроек pyrightconfig.json

## Обзор

Этот файл содержит подробное описание всех параметров конфигурации линтера `basedpyright` (pyright) для проекта FSA-AstraInstall.

---

## Исключения из проверки

### `exclude`
Список директорий и файлов, которые исключены из проверки типов:
- `**/node_modules` - Node.js модули (не нужны для Python проекта)
- `**/__pycache__` - Кэш Python байт-кода
- `**/.git` - Git репозиторий
- `**/build`, `**/dist` - Директории сборки и дистрибутивов
- `**/*.pyc` - Скомпилированные Python файлы
- `**/.pytest_cache`, `**/.mypy_cache` - Кэши тестирования
- `**/History`, `**/TestScripts`, `**/DocPlanes`, `**/DocInstruction` - Документация и история
- `**/DockerManager`, `**/PackageManager`, `**/WineTricks`, `**/RunScript` - Модули проекта
- `**/Screenshots`, `**/Icons`, `**/*.pdf` - Медиа файлы
- `FSA-AstraInstall-1-7`, `FSA-AstraInstall-1-8` - Старые версии проекта

## Основные настройки проверки

### `reportMissingImports: true`
**ВКЛЮЧЕНО** - Проверка отсутствующих импортов
- **Что проверяет**: Импорты модулей, которые не могут быть найдены
- **Важность**: Высокая - помогает обнаружить ошибки на этапе разработки

### `reportMissingTypeStubs: false`
**ОТКЛЮЧЕНО** - Проверка отсутствующих type stubs
- **Что проверяет**: Наличие файлов с аннотациями типов для внешних библиотек
- **Важность**: Низкая - не критично для работы приложения

## ⚠️ КРИТИЧЕСКИ ВАЖНЫЕ ПРОВЕРКИ (ОТКЛЮЧЕНЫ)

**ВНИМАНИЕ**: Эти проверки отключены, но они могут обнаруживать реальные проблемы!
Рекомендуется периодически включать их для проверки кода.

### `reportAttributeAccessIssue: "none"`
**ОТКЛЮЧЕНО** - Проверка доступа к атрибутам объектов
- **Что проверяет**: Попытки доступа к несуществующим атрибутам или методам
- **Пример ошибки**: `obj.non_existent_method()` где метод не существует
- **Почему отключено**: Для динамических атрибутов (`sys._gui_instance`, `builtins._original_print`)
- **⚠️ РИСК**: Может скрывать реальные ошибки доступа к несуществующим атрибутам (AttributeError)

### `reportCallIssue: "none"`
**ОТКЛЮЧЕНО** - Проверка вызовов функций
- **Что проверяет**: Неправильные вызовы функций (неверное количество аргументов, типы)
- **Пример ошибки**: `func(arg1, arg2)` где функция принимает только 1 аргумент
- **Почему отключено**: Для переопределенного `print()` с дополнительными параметрами
- **⚠️ РИСК**: Может скрывать реальные ошибки вызовов функций (TypeError)

### `reportOptionalMemberAccess: "none"`
**ОТКЛЮЧЕНО** - Проверка доступа к членам Optional типов
- **Что проверяет**: Доступ к атрибутам/методам переменных, которые могут быть `None`
- **Пример ошибки**: 
  ```python
  obj: Optional[MyClass] = get_object()
  obj.method()  # Ошибка: obj может быть None
  ```
- **⚠️ КРИТИЧЕСКИЙ РИСК**: Может скрывать `AttributeError` при обращении к `None`
- **Решение**: Всегда проверять на `None` перед доступом: `if obj is not None: obj.method()`

### `reportPossiblyUnboundVariable: "none"`
**ОТКЛЮЧЕНО** - Проверка неинициализированных переменных
- **Что проверяет**: Использование переменных до их инициализации
- **Пример ошибки**: 
  ```python
  if flag:
      x = 1
  return x  # Ошибка: x может быть не определена если flag=False
  ```
- **⚠️ КРИТИЧЕСКИЙ РИСК**: Может скрывать `NameError` при использовании неинициализированных переменных
- **Решение**: Инициализировать переменные перед условными блоками

### `reportOptionalSubscript: "none"`
**ОТКЛЮЧЕНО** - Проверка индексации Optional типов
- **Что проверяет**: Индексация списков/словарей, которые могут быть `None`
- **Пример ошибки**: 
  ```python
  lst: Optional[List[int]] = get_list()
  value = lst[0]  # Ошибка: lst может быть None
  ```
- **⚠️ КРИТИЧЕСКИЙ РИСК**: Может скрывать `TypeError` при индексации `None`
- **Решение**: Проверять на `None`: `if lst is not None: value = lst[0]`

### `reportOptionalOperand: "none"`
**ОТКЛЮЧЕНО** - Проверка операций с Optional типами
- **Что проверяет**: Арифметические/логические операции с переменными, которые могут быть `None`
- **Пример ошибки**: 
  ```python
  x: Optional[int] = get_value()
  result = x + 1  # Ошибка: x может быть None
  ```
- **⚠️ КРИТИЧЕСКИЙ РИСК**: Может скрывать `TypeError` при операциях с `None`
- **Решение**: Проверять на `None`: `if x is not None: result = x + 1`

### `reportOptionalIterable: "none"`
**ОТКЛЮЧЕНО** - Проверка итерации по Optional типам
- **Что проверяет**: Использование в циклах `for` переменных, которые могут быть `None`
- **Пример ошибки**: 
  ```python
  lst: Optional[List[int]] = get_list()
  for item in lst:  # Ошибка: lst может быть None
      print(item)
  ```
- **⚠️ КРИТИЧЕСКИЙ РИСК**: Может скрывать `TypeError` при итерации по `None`
- **Решение**: Проверять на `None`: `if lst is not None: for item in lst: ...`

### `reportOptionalContextManager: "none"`
**ОТКЛЮЧЕНО** - Проверка использования Optional как контекстных менеджеров
- **Что проверяет**: Использование `with` для переменных, которые могут быть `None`
- **Пример ошибки**: 
  ```python
  f: Optional[File] = get_file()
  with f:  # Ошибка: f может быть None
      f.read()
  ```
- **⚠️ КРИТИЧЕСКИЙ РИСК**: Может скрывать `AttributeError` при использовании `None` в `with`
- **Решение**: Проверять на `None`: `if f is not None: with f: ...`

### `reportOptionalCall: "none"`
**ОТКЛЮЧЕНО** - Проверка вызова Optional функций
- **Что проверяет**: Вызов функций, которые могут быть `None`
- **Пример ошибки**: 
  ```python
  func: Optional[Callable] = get_function()
  func()  # Ошибка: func может быть None
  ```
- **⚠️ КРИТИЧЕСКИЙ РИСК**: Может скрывать `TypeError` при вызове `None` как функции
- **Решение**: Проверять на `None`: `if func is not None: func()`

### `reportOperatorIssue: "none"`
**ОТКЛЮЧЕНО** - Проверка операторов
- **Что проверяет**: Неправильное использование операторов (+, -, *, /, ==, и т.д.)
- **Пример ошибки**: Операции между несовместимыми типами
- **⚠️ РИСК**: Может скрывать ошибки типов в операциях
- **Решение**: Убедиться, что типы совместимы для операций

## Проверки с предупреждениями (не блокируют работу)

### `reportGeneralTypeIssues: "warning"`
**ВКЛЮЧЕНО КАК ПРЕДУПРЕЖДЕНИЕ** - Общие проблемы с типами
- **Что проверяет**: Общие несоответствия типов
- **Уровень**: Предупреждение (не блокирует работу)

### `reportArgumentType: "warning"`
**ВКЛЮЧЕНО КАК ПРЕДУПРЕЖДЕНИЕ** - Проверка типов аргументов
- **Что проверяет**: Несоответствие типов аргументов функции ожидаемым типам
- **Пример**: Передача `str` вместо `int` в функцию
- **Уровень**: Предупреждение (не блокирует работу)

### `reportReturnType: "warning"`
**ВКЛЮЧЕНО КАК ПРЕДУПРЕЖДЕНИЕ** - Проверка типов возвращаемых значений
- **Что проверяет**: Несоответствие возвращаемого значения заявленному типу
- **Пример**: Функция объявлена как `-> str`, но возвращает `None`
- **Уровень**: Предупреждение (не блокирует работу)

### `reportUnusedExcept: "warning"`
**ВКЛЮЧЕНО КАК ПРЕДУПРЕЖДЕНИЕ** - Неиспользуемые исключения
- **Что проверяет**: Перехват исключений, которые не используются в блоке `except`
- **Пример**: `except Exception as e:` где `e` не используется
- **Уровень**: Предупреждение (не блокирует работу)

## Проверки неиспользуемого кода (отключены - не влияют на работоспособность)

- **Что проверяет**: Классы, которые определены, но нигде не используются
- **Важность**: Низкая - не влияет на работоспособность, только на чистоту кода

### `reportUnusedFunction: "none"`

- **Что проверяет**: Функции, которые определены, но нигде не вызываются
- **Важность**: Низкая - не влияет на работоспособность, только на чистоту кода

**ОТКЛЮЧЕНО** - Проверка неиспользуемых переменных
- **Что проверяет**: Переменные, которые объявлены, но не используются
- **Важность**: Низкая - не влияет на работоспособность, только на чистоту кода

### `reportUnusedImport: "none"`
**ОТКЛЮЧЕНО** - Проверка неиспользуемых импортов
- **Что проверяет**: Импорты модулей, которые не используются в коде
- **Важность**: Низкая - не влияет на работоспособность, только на чистоту кода

## Режим проверки типов

### `typeCheckingMode: "basic"`
**РЕЖИМ**: Базовый (менее строгий)
- **Варианты**: 
  - `"off"` - выключено
  - `"basic"` - базовый (текущий)
  - `"standard"` - стандартный
  - `"strict"` - строгий
- **Текущий режим**: `"basic"` - минимальные проверки, совместим с динамическим Python кодом
- **Рекомендация**: Для более строгой проверки можно переключить на `"standard"` или `"strict"`

---

## Рекомендации

1. **Периодически включать критичные проверки**: Время от времени включайте `reportOptionalMemberAccess`, `reportPossiblyUnboundVariable` и другие для проверки кода на реальные ошибки.

2. **Исправлять предупреждения**: Обращайте внимание на предупреждения (`warning`) - они указывают на потенциальные проблемы.

3. **Проверять перед коммитом**: Перед коммитом можно временно включить строгие проверки для обнаружения проблем.

4. **Использовать type hints**: Добавляйте аннотации типов в код для лучшей проверки.

---

## Примеры включения проверок для тестирования

Временно измените значения на `"warning"` или `"error"` для проверки:

```json
{
  "reportOptionalMemberAccess": "warning",
  "reportPossiblyUnboundVariable": "warning",
  "reportOptionalSubscript": "warning"
}
```

После проверки верните обратно на `"none"` если нужно.
