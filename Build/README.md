## 📋 Описание

Эта папка содержит скрипты для сборки проекта FSA-AstraInstall.

## 📁 Содержимое

### `generate_unified.py`
Скрипт генерации объединенного файла для сборки.

**Назначение:** Генерирует `FSA-AstraInstall.py` из исходников проекта.

**Использование:**
```bash
python3 Build/generate_unified.py
```

**Что делает:**
- Вызывает `build_unified.py` для создания объединенного файла
- Проверяет результат
- Выводит информацию о созданном файле

### `build_unified.py`
Скрипт объединения файлов проекта в единый Python файл.

**Назначение:** Объединяет все файлы проекта в `FSA-AstraInstall.py`.

**Что объединяет:**
1. `astra_automation.py` - основной модуль (из корня проекта)
2. `astra_update.sh` - скрипт обновления (из корня проекта, встраивается как строка)
3. `astra_install.sh` - скрипт установки (из корня проекта, встраивается как строка)
4. `self_updater.py` - модуль самообновления (из папки Build/)

### `self_updater.py`
Модуль самообновления для FSA-AstraInstall.

**Назначение:** Проверяет наличие обновлений и заменяет текущий файл новым.

**Функционал:**
- Проверка обновлений через SMB и Git
- Скачивание новой версии
- Замена текущего файла
- Работает для бинарной и Python версий

**Использование:** Встраивается в `FSA-AstraInstall.py` через `build_unified.py`

**Запуск build_unified.py:**
```bash
python3 Build/build_unified.py
```

**Результат:** Создается `FSA-AstraInstall.py` в корне проекта.

## 🔄 Процесс сборки

1. **Генерация объединенного файла:**
   ```bash
   python3 Build/generate_unified.py
   ```
   Результат: `FSA-AstraInstall.py` в корне проекта

2. **Сборка бинарника (через DockerManager):**
   ```bash
   python3 -m DockerManager.cli --project FSA-AstraInstall --platform astra-1.8
   ```
   Результат: `FSA-AstraInstall` (бинарный файл) в корне проекта

## 📝 Примечания

- Исходные файлы **НЕ изменяются** - все изменения только в выходных файлах
- Результат создается в **корне проекта**, а не в папке Build/
- Скрипты автоматически определяют пути к файлам в корне проекта
