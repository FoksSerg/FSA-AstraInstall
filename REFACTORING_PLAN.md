## 🏗️ Структура классов

### 1. Класс `ComponentManager`

**Назначение:** Управление компонентами и их конфигурацией

**Методы (8):**
1. `expand_user_path(path)` - расширение пути с учетом SUDO_USER
2. `get_component_field(component_id, field_name, default=None)` - получение поля компонента
3. `get_component_data(component_id)` - получение всей конфигурации компонента
4. `check_component_status(component_id, wineprefix_path=None)` - проверка статуса компонента
5. `resolve_dependencies(component_id)` - разрешение зависимостей компонента
6. `resolve_dependencies_for_install(component_ids)` - разрешение зависимостей для установки
7. `resolve_dependencies_for_uninstall(component_ids)` - разрешение зависимостей для удаления
8. `validate_component_config()` - валидация конфигурации компонентов

**Зависимости:**
- Использует глобальный `COMPONENTS_CONFIG`
- Методы могут вызывать друг друга через `self.method_name()`

**Глобальный экземпляр:**
```python
_component_manager = ComponentManager()
```

**Использование:**
- В методах классов: `_component_manager.get_component_field(...)`
- В обычных функциях: `_component_manager.get_component_field(...)`
- Внутри методов `ComponentManager`: `self.get_component_field(...)`

---

### 2. Класс `SystemUtils`

**Назначение:** Системные утилиты и проверки

**Методы (6):**
1. `sync_system_time()` - синхронизация системного времени
2. `check_system_requirements()` - проверка системных требований
3. `install_gui_components()` - установка GUI компонентов (tkinter, pip3)
4. `cleanup_temp_files(temp_dir)` - очистка временных файлов
5. `find_running_instance()` - поиск запущенного экземпляра программы
6. `activate_existing_window()` - активация существующего окна

**Глобальный экземпляр:**
```python
_system_utils = SystemUtils()
```

**Использование:**
- В `main()`: `_system_utils.check_system_requirements()`
- В других местах: `_system_utils.sync_system_time()`
