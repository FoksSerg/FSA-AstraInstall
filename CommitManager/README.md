# CommitManager

Универсальный инструмент для создания коммитов с поддержкой настройки для разных проектов.

**Версия:** V3.4.184 (2025.12.16)  
**Компания:** ООО "НПА Вира-Реалтайм"  
**Разработчик:** @FoksSegr & AI Assistant (@LLM)

## 📋 Описание

CommitManager - это GUI инструмент для автоматизации создания коммитов с соблюдением правил версионирования и создания истории. Модуль предоставляет:

- ✅ Автоматическое выполнение алгоритма создания коммитов (21 шаг)
- ✅ Настройка для разных проектов
- ✅ Управление ключевыми файлами и историей
- ✅ Интеграция с AI для анализа изменений
- ✅ Визуальный контроль процесса выполнения
- ✅ Поддержка пауз и подтверждений

## 📁 Структура модуля

### Основные компоненты

1. **ConfigManager** - Менеджер конфигураций:
   - Загрузка/сохранение конфигураций проектов
   - Управление текущим проектом
   - Создание конфигураций по умолчанию

2. **ProjectConfig** - Конфигурация проекта:
   - Настройки версионирования
   - Ключевые файлы
   - Настройки истории
   - Настройки AI анализа

3. **CommitExecutor** - Исполнитель алгоритма:
   - Выполнение всех 21 шага алгоритма
   - Управление состоянием процесса
   - Интеграция с git командами

4. **CommitAnalyzer** - Анализатор изменений:
   - Анализ через AI API (OpenAI/Anthropic)
   - Гибридный режим (ожидание ассистента)
   - Локальная LLM (планируется)
   - Ручной режим

5. **CommitManagerGUI** - GUI интерфейс:
   - Вкладка "Процесс" - выполнение алгоритма
   - Вкладка "Настройки проекта" - конфигурация проекта
   - Вкладка "Ключевые файлы" - управление ключевыми файлами
   - Вкладка "История" - настройки истории
   - Вкладка "AI Анализ" - настройки AI

## 📝 Использование

### Создание коммита

1. Запустите GUI: `python3 -m CommitManager.run_gui`
2. Выберите проект (или используйте текущий)
3. Нажмите "▶ Старт" на вкладке "Процесс"
4. Следите за выполнением шагов
5. При необходимости подтвердите действия на паузах

### Настройка проекта

1. Откройте вкладку "Настройки проекта"
2. Укажите путь к проекту и файл версии
3. Настройте правила версионирования
4. Сохраните настройки

### Управление ключевыми файлами

1. Откройте вкладку "Ключевые файлы"
2. Добавьте/удалите ключевые файлы
3. Настройте бинарные файлы для пересборки
4. Сохраните изменения
