# Инструкция по тестированию winetricks_tracker.py

## Быстрый тест

Для тестирования на простом компоненте `cabinet` (Microsoft cabinet.dll):

```bash
cd /Volumes/FSA-PRJ/Project/FSA-AstraInstall/WineTricks

# Запуск установки с отслеживанием
python3 winetricks_tracker.py cabinet
```

## Что делает скрипт:

1. **Захватывает состояние ДО установки:**
   - Сканирует файловую систему (system32, syswow64, Fonts)
   - Экспортирует реестр
   - Сохраняет DLL overrides

2. **Запускает winetricks** для установки компонента

3. **Захватывает состояние ПОСЛЕ установки:**
   - Снова сканирует файловую систему
   - Экспортирует реестр
   - Сохраняет DLL overrides

4. **Анализирует изменения:**
   - Находит новые файлы
   - Находит измененные файлы
   - Находит новые DLL overrides

5. **Генерирует:**
   - Манифест установки (JSON)
   - Скрипт деинсталляции (bash)

## Результаты

После выполнения в папке `install_logs/cabinet/` будут созданы:

- `cabinet_manifest.json` - полный манифест установки
- `registry_before.reg` - реестр до установки
- `registry_after.reg` - реестр после установки
- `uninstall_cabinet.sh` - скрипт деинсталляции

## Тестирование деинсталляции

После установки можно протестировать деинсталляцию:

```bash
# Запуск скрипта деинсталляции
bash install_logs/cabinet/uninstall_cabinet.sh
```

## Другие компоненты для теста

Простые компоненты для тестирования:
- `cabinet` - Microsoft cabinet.dll (простая DLL)
- `cmd` - MS cmd.exe (один файл)
- `binkw32` - RAD Game Tools binkw32.dll

Более сложные (для полного теста):
- `vcrun2019` - Visual C++ Runtime (много файлов)
- `dotnet48` - .NET Framework (большая установка)

## Примечания

- Убедитесь, что у вас настроен WINEPREFIX (по умолчанию ~/.wine)
- Скрипт требует права на чтение/запись в WINEPREFIX
- Для компонентов, которые уже установлены, используйте `--force`
