Автоматический перезапуск через sudo и закрытие терминала после запуска GUI

Изменения:
- Bash скрипт автоматически перезапускается через sudo если запущен без прав root
- GUI автоматически закрывает окно родительского терминала через 2 секунды после запуска
- Передача PID окна терминала (не bash скрипта) для корректного закрытия
- Обновлены инструкции в README - теперь не нужно явно указывать sudo

Bash скрипт (astra_install.sh):
- Проверка прав root с автоматическим exec sudo -E bash "$0" "$@"
- Определение PID окна терминала через ps -o ppid= -p $PPID
- Передача PID в GUI через --close-terminal параметр
- Упрощенная логика запуска GUI

Python GUI (astra-automation.py):
- Новый параметр --close-terminal PID для автозакрытия терминала
- Метод _close_parent_terminal() с отправкой SIGTERM/SIGKILL
- Запланированное закрытие через root.after(2000, ...)
- Обработка ошибок и логирование операции закрытия

README:
- Обновлены примеры запуска: bash astra_install.sh (без sudo)
- Добавлено примечание об автоматическом запросе прав
- Обновлены все примеры для консольного и dry-run режимов

Преимущества:
- Удобнее использование - не нужно помнить про sudo
- GUI работает независимо от терминала (nohup + автозакрытие)
- Чистый рабочий стол - окно терминала не остается висеть
- Корректная работа sudo с сохранением переменных окружения (-E)
