## 🔍 Анализ текущей ситуации

### Текущая архитектура

**Текущая структура запуска:**
```
astra_install.sh (bash скрипт)
    ↓
    ├─ Проверка прав root → sudo перезапуск
    ├─ Проверка репозиториев
    ├─ Установка системных пакетов (apt-get):
    │   ├─ python3-tk (tkinter)
    │   ├─ python3-psutil
    │   ├─ wmctrl
    │   ├─ xdotool
    │   └─ expect
    ├─ Запуск Python скрипта:
    │   └─ astra_automation.py (19,061 строк)
    │       ├─ GUI (tkinter)
    │       ├─ Управление процессами
    │       ├─ Установка Wine
    │       └─ Установка компонентов
```

### Зависимости проекта

#### Системные зависимости (устанавливаются через apt-get):
- **python3-tk** - библиотека tkinter для GUI
- **python3-psutil** - мониторинг системы
- **wmctrl** - управление окнами (single instance)
- **xdotool** - управление окнами (single instance)
- **expect** - автоматизация интерактивных процессов

#### Python зависимости (встроенные в стандартную библиотеку):
- `os`, `sys`, `subprocess`, `threading`, `queue`
- `tempfile`, `shutil`, `glob`, `platform`
- `signal`, `datetime`, `hashlib`, `re`
- `abc`, `types`, `ast`, `builtins`, `textwrap`
- `collections.deque`

#### Python зависимости (опциональные, могут отсутствовать):
- **psutil** - мониторинг системы (устанавливается через apt-get)
- **requests** - HTTP клиент (опционально)
- **tkinter** - GUI библиотека (системный пакет)

#### Внешние ресурсы:
- **AstraPack/** - архивы с пакетами:
  - `AstraPack/Astra/astra_packages.tar.gz`
  - `AstraPack/Cont/CountPack.tar.gz`
  - `AstraPack/Wine/wine_packages.tar.gz`
  - `AstraPack/Winetricks/` - скрипты winetricks

### Проблемы текущей архитектуры

1. **Два файла для запуска:**
   - `astra_install.sh` - bash скрипт (766 строк)
   - `astra_automation.py` - Python скрипт (19,061 строк)

2. **Зависимость от bash скрипта:**
   - Логика установки пакетов в bash
   - Определение режима запуска в bash
   - Передача параметров между bash и Python

3. **Системные пакеты не упаковываются:**
   - tkinter - системная библиотека
   - wmctrl, xdotool - системные утилиты
   - Нельзя упаковать в исполняемый файл

4. **Консольный режим:**
   - Требует отдельной обработки
   - Может конфликтовать с single instance проверкой

## 🎯 Варианты реализации

### Вариант 1: Полностью автономный исполняемый файл (Pure Python)

**Описание:** Один исполняемый файл, который сам проверяет и устанавливает все системные зависимости.

**Архитектура:**
```
astra_install (исполняемый файл)
    ↓
    ├─ Проверка прав root → запрос sudo
    ├─ Проверка системных пакетов
    ├─ Установка недостающих пакетов (apt-get)
    ├─ Запуск GUI/консольного режима
    └─ Вся логика в одном файле
```

**Преимущества:**
- ✅ Один файл - простота распространения
- ✅ Не требует bash скрипта
- ✅ Вся логика в Python
- ✅ Легко обновлять

**Недостатки:**
- ❌ Требует прав root для установки пакетов
- ❌ Зависит от apt-get (только Linux)
- ❌ Первый запуск может быть медленным (установка пакетов)
- ❌ Нужно запрашивать пароль sudo

**Технические детали:**
- PyInstaller собирает один файл
- Встроенная логика установки пакетов
- Проверка и установка при каждом запуске (или кеширование)

```python
def check_system_packages(packages):
    """Проверка наличия системных пакетов"""
    missing = []
    for pkg in packages:
        if not check_package_installed(pkg):
            missing.append(pkg)
    return missing

def check_package_installed(package):
    """Проверка установлен ли пакет через dpkg"""
    try:
        result = subprocess.run(
            ['dpkg', '-l', package],
            capture_output=True,
            text=True,
            timeout=2
        )
        return result.returncode == 0 and package in result.stdout
    except:
        return False

def install_system_packages(packages):
    """Установка системных пакетов через apt-get"""
    env = os.environ.copy()
    env['DEBIAN_FRONTEND'] = 'noninteractive'
    env['DEBIAN_PRIORITY'] = 'critical'
    env['APT_LISTCHANGES_FRONTEND'] = 'none'
    
    cmd = [
        'apt-get', 'update', '-y'
    ]
    subprocess.run(cmd, env=env, check=True)
    
    cmd = [
        'apt-get', 'install', '-y',
        '-o', 'Dpkg::Options::=--force-confdef',
        '-o', 'Dpkg::Options::=--force-confnew',
        '-o', 'Dpkg::Options::=--force-confmiss'
    ] + packages
    
    subprocess.run(cmd, env=env, check=True)
```

**Минимальный `astra_install.sh`:**
```bash
#!/bin/bash
# Минимальный скрипт для установки системных пакетов

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
cd "$SCRIPT_DIR"

# Проверка прав root
if [ "$EUID" -ne 0 ]; then
    exec sudo "$0" "$@"
fi

# Список необходимых пакетов
REQUIRED_PACKAGES=(
    "python3-tk"
    "python3-psutil"
    "wmctrl"
    "xdotool"
    "expect"
)

# Проверка и установка пакетов
MISSING_PACKAGES=()
for pkg in "${REQUIRED_PACKAGES[@]}"; do
    if ! dpkg -l | grep -q "^ii  $pkg "; then
        MISSING_PACKAGES+=("$pkg")
    fi
done

if [ ${#MISSING_PACKAGES[@]} -gt 0 ]; then
    echo "[INFO] Установка недостающих пакетов..."
    export DEBIAN_FRONTEND=noninteractive
    apt-get update -y
    apt-get install -y "${MISSING_PACKAGES[@]}"
fi

# Запуск исполняемого файла
exec "$SCRIPT_DIR/astra_install" "$@"
```

- Содержит всю логику приложения
- Не устанавливает системные пакеты
- Предполагает, что пакеты уже установлены

```python
def main():
    """Главная функция с умной установкой пакетов"""

            'python3-tk',
            'python3-psutil',
            'wmctrl',
            'xdotool',
            'expect'

        missing_packages = check_system_packages(required_packages)
        if missing_packages:
            print(f"[INFO] Установка недостающих пакетов: {', '.join(missing_packages)}")
            install_system_packages(missing_packages)
        
        # ШАГ 4: Проверка tkinter
        if not check_tkinter_available():
            print("[ERROR] tkinter не доступен после установки")
            sys.exit(1)

            f.write(f"Установлено: {datetime.datetime.now()}\n")
    
    # ШАГ 6: Запуск приложения
    # ... существующая логика ...
```

**Оптимизация проверки:**
```python
def check_system_packages_fast(packages):
    """Быстрая проверка через кеш dpkg"""
    cache_file = '/tmp/astra_install_packages_cache'
    
    # Читаем кеш (если есть)
    if os.path.exists(cache_file):
        with open(cache_file, 'r') as f:
            cached_packages = set(f.read().splitlines())
            missing = [pkg for pkg in packages if pkg not in cached_packages]
            if not missing:
                return []  # Все пакеты установлены
    
    # Полная проверка
    missing = check_system_packages(packages)
    
    # Обновляем кеш
    if not missing:
        with open(cache_file, 'w') as f:
            f.write('\n'.join(packages))
    
    return missing
```

```bash
# Проверка и установка системных пакетов
if [ "$EUID" -ne 0 ]; then
    # Запускаем с sudo для установки пакетов
    sudo "$APPDIR/usr/bin/install_packages.sh"
fi

# Запуск приложения
exec "$APPDIR/usr/bin/astra_install" "$@"
```

```bash
#!/bin/bash
```

## ⚠️ Риски и их митигация

### Риск 1: Отсутствие прав root

**Проблема:** Исполняемый файл требует прав root для установки пакетов.

**Митигация:**
- Автоматический перезапуск с sudo
- Четкое сообщение пользователю о необходимости прав root
- Возможность запуска в консольном режиме без установки пакетов

**Код:**
```python
if os.geteuid() != 0:
    print("[INFO] Требуются права root для установки системных пакетов")
    print("[INFO] Перезапуск с sudo...")
    subprocess.call(['sudo', sys.executable] + sys.argv)
    sys.exit(0)
```

### Риск 2: Отсутствие apt-get

**Проблема:** Система не использует apt-get (например, RPM-based системы).

**Митигация:**
- Проверка наличия apt-get перед установкой
- Сообщение об ошибке с инструкциями
- Возможность пропуска установки пакетов

**Код:**
```python
def check_apt_get_available():
    """Проверка наличия apt-get"""
    try:
        subprocess.run(['which', 'apt-get'], 
                     capture_output=True, 
                     check=True)
        return True
    except:
        return False

if not check_apt_get_available():
    print("[ERROR] apt-get не найден")
    print("[INFO] Эта система не поддерживается")
    sys.exit(1)
```

### Риск 3: Ошибки установки пакетов

**Проблема:** Установка пакетов может завершиться с ошибкой.

**Митигация:**
- Обработка всех ошибок установки
- Логирование ошибок
- Продолжение работы, если возможно
- Сообщение пользователю о проблемах

**Код:**
```python
def install_system_packages(packages):
    """Установка с обработкой ошибок"""
    try:
        # Обновление списков
        subprocess.run(['apt-get', 'update', '-y'], check=True)
        
        # Установка пакетов
        subprocess.run(['apt-get', 'install', '-y'] + packages, check=True)
        return True
    except subprocess.CalledProcessError as e:
        print(f"[ERROR] Ошибка установки пакетов: {e}")
        return False
    except Exception as e:
        print(f"[ERROR] Неожиданная ошибка: {e}")
        return False
```
